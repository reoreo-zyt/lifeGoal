# 征召兵系统设计

> 本文档定义游戏中「征召兵」的设计规范，包括概念、用途、上限机制及与现有系统的交互。

---

## 概念

征召兵是玩家在跑图过程中积累的**全局兵力储备**，用于为我方上阵武将分配兵力。没有征召兵则无法为武将分配兵力，兵力为零的武将无法参战。

**核心定位**：征召兵是"全局弹药池"，而非武将自身属性。玩家通过商店购买、事件奖励等方式获取征召兵，在**上阵时**或**战斗前**分配给武将。

---

## 数值设计

| 参数 | 值 | 说明 |
|------|-----|------|
| 最大征召兵上限 | 3000 | 全局兵力储备上限 |
| 初始征召兵 | 3000 | 游戏开始时的起始兵力 |
| 单次分配单位 | 无限制 | 可按需任意分配，最小 1 |

### 获取途径

| 来源 | 数量 | 说明 |
|------|------|------|
| 商店购买 | 300~500/次 | 消耗 180 金币 |
| 事件奖励 | 200~800 | 随机事件选项 |

### 分配时机

征召兵可在以下两个时机分配给武将：

1. **上阵时分配**：点击武将槽位选择武将后，可分配征召兵给该武将
2. **战前统一分配**：战斗开始前若有武将兵力不足，可统一分配

### 消耗场景

- **为武将分配兵力**：从征召兵池扣除，分配给上阵武将
- **战斗中兵力损失**：从武将当前兵力扣除，征召兵池不变
- **武将阵亡**：武将下阵，已分配的兵力不回收

## UI 设计

### 状态栏展示

顶部状态栏显示：
```
征召兵：800 / 3000  [━━━━━━░░░░]
```

- 显示当前值 / 最大上限
- 进度条直观展示剩余比例
- 点击可查看详细说明

### 上阵时分配

点击武将槽位后：
- 左侧：武将信息（名称、兵力条）
- 右侧：征召兵分配滑块
  - 显示：可分配数量
  - 滑动分配
  - 实时预览：该武将将获得的兵力

### 战前统一分配

战斗开始前检测：
- 若有武将兵力为 0，显示提示：「部分武将兵力为 0，请分配征召兵」
- 弹出简洁分配界面，快速为兵力不足的武将补充

### 事件奖励展示（EventPanel）

选择界面中，征召兵效果标签：
```
+300 征召兵
```

## 商店逻辑

```typescript
const resolveShopNode = () => {
  if (money.value >= 180) {
    money.value -= 180;
    // 消耗180金币，补充征召兵
    const added = Math.min(500, maxConscripts - totalConscripts);
    totalConscripts += added;
    addReport(`商店交易：花费 180 金币，补充征召兵 +${added}。`);
  }
};
```

## 与旧系统的区别（对比已删除的旧版）

| 维度 | 旧版（已删除） | 新版 |
|------|--------------|------|
| 分配方式 | 战斗中实时分配 | 上阵时/战前分配 |
| 兵力来源 | 武将自己兵力 | 全局弹药池 |
| 消耗方式 | 战斗损失直接扣 | 分配时扣，战斗损失不涉及征召兵 |
| 上限 | 无上限 | 有全局上限 3000 |
| 恢复 | Boss战/休息恢复自身兵力 | 征召兵通过商店/事件恢复 |

## 实施要点

1. **在 `game.ts` 的 `VictoryRewardOption` 中添加 `conscript` 类型**
2. **在 `event-types.ts` 的 `EffectType` 中添加 `conscript`**
3. **在 `CardBattleGame.vue` 添加全局 `totalConscripts` ref（初始 1000，上限 3000）**
4. **修改商店节点：消耗 180 金币补充 300~500 征召兵**
5. **事件效果支持 `conscript` 类型（恢复征召兵）**
6. **UI 状态栏显示征召兵进度条**
7. **上阵/战前分配界面支持从征召兵池分配给武将**

## 后续扩展方向

- **高级商店**：出售更高数量的征召兵（消耗更多金币）
- **征召兵相关遗物**：如「辎重车」可增加征召兵上限或获取量
- **溢出兑换**：征召兵满后溢出时可选择兑换少量金币
