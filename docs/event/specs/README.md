# 地图随机事件系统设计

> 版本：1.0
> 日期：2026-04-30
> 状态：待评审

---

## 概述

本系统为 rogue-like 地图设计叙事分支事件。每个事件包含一段历史典故文本和两个选项，玩家选择后将影响后续事件池、属性状态和游戏进程。

### 设计原则

- **史实为骨**：所有事件文本引用正史（《旧唐书》《新唐书》《资治通鉴》等）
- **代入为本**：二选一机制让玩家置身历史抉择之中
- **因果闭环**：选项会永久改变后续事件池，事件之间存在因果逻辑

## 事件类型定义

```typescript
// src/events/event-types.ts

export type EventChapter = "sui" | "chen" | "tang" | "tian" | "zhen" | "hun";

export interface Effect {
  type: "gold" | "conscript" | "attack" | "defense" | "speed" | "strategy"
       | "attributePercent" | "moral" | "specialBuff" | "specialDebuff";
  value: number;
  target?: "all" | "general";
  duration?: number;  // 持续回合数，不填为永久
  buffId?: string;
}

export interface EventChoice {
  label: string;       // 选项标题
  description: string; // 选项详细描述
  effects: Effect[];   // 即时效果
  disablesEvents?: string[];  // 选择后禁用的其他事件ID
  enablesEvents?: string[];   // 选择后解锁的事件ID
  flags?: string[];          // 设置的全局标记
}

export interface MapEvent {
  id: string;
  title: string;
  narrative: string;        // 历史典故文本（引用正史）
  narrativeSource: string;   // 出处，如《资治通鉴》卷一九一
  chapter: EventChapter;
  choiceA: EventChoice;
  choiceB: EventChoice;
  requiredGenerals: string[];
  weightBonus: Record<string, number>;
  disabled?: boolean;
}
```

## 触发机制

### 动态权重系统

```
基础权重 = 1.0

加成规则：
- 拥有 requiredGenerals 中任意一个：+0.5
- 拥有完整组合羁绊：+1.0
- 拥有对应武将数量 > 2：+0.5
- 全局标记触发：按标记加成

减益规则：
- 被禁用事件的关联事件：-0.8
- 未拥有任何相关武将：-0.5
```

### 抽取逻辑

```
当玩家进入"event"类型节点时：
1. 扫描玩家当前拥有的武将列表
2. 收集当前激活的全局标记（flags）
3. 过滤事件池，移除：
   - disabled=true 的事件
   - 未满足 requiredGenerals 的事件（权重降为0）
   - 被其他选项禁用了的事件
4. 从候选池中按权重随机抽取 1 个事件展示
```
